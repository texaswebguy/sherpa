Creating a Theme

The Sherpa Starter Kit comes with the default Sherpa css theme.

CSS themes can be built using less and provide control of global elements such as colors, heights, widths, paddings, etc.

In addition, some component level variables are provided to make theme adjustments.

If further customization is provided, a theme can also declarations that override Sherpa classes or a whole new non sherpa css can be specified.

There are three ways to specify or switch a theme:

  • Configuration of default theme
  • In the url
  • Via JavaScript

Configuration of default theme:

By editing assets/config/global_config.js you can change the default theme.

Note this line contains the default them css to be loaded:
{ css_theme: SHERPA.CSS_CORE_PATH+"sherpa.css" }

Switch theme through url:

You might want to provide a link on your project index page that would switch the theme.

For example: See foobar theme. Reset theme.

Via JavaScript:

You can have any function execute this to initiate a page reload with the new theme.

Sherpa.themeSwitcher("foobar");

Special classes for theme logic:

If you create a theme called "foobar" you have two classes available that will provide show/hide logic capability:

  • theme-hide-foobar
  • theme-only-foobar

Note: this uses a display: block; so you can't really use it in an in-line display (word in a middle of a sentence).

Markup:

<div class="theme-hide-foobar">
	This element will only display only when any theme other than "foobar" is in context.
</div>
<div class="theme-only-foobar">
	This element will only display when the "foobar" theme is in context.
</div>
