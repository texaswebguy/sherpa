Global generic modal

The easiest way to create modals is to leverage a Sherpa configuration that enables a global generic modal that is ready to use at any time (on by default). In assets/config/global_config.js you will see SHERPA.ENABLE_GLOBAL_MODAL = true;. This enables the global generic modal that can be activated via javascript or through html attributes.

Activating modals

Via data attributes

If you want to activate a custom hiden modal without writing JavaScript you can set data-toggle="modal" on a controller element, like a button, along with a data-target="#foo" or href="#foo" to target a specific modal to toggle.

  1. <button type="button" data-toggle="modal" data-target="#myModal">Launch modal</button>

Activating the generic modal

To activate a the global generic modal without writing JavaScript you can set data-toggle="modal" on a controller element, like a button. The generic modal is devided into three distinct areas, header, body and footer.

You can provide a title and title description via attributes by using data-title="This is the title of the modal" and data-titledescription="description of title" respectively. Title and description attributes are optional but the title is required if description is used.

For the body of the modal you can provide text via data-body="This is text that will be inserted in the body of the modal" or you can also inject the contents of a hidden html element (div) via selector such as data-bodyhtml="#some_id".

The footer of the modal works exactly as the body using data-body="<a href='#' class='btn'>Continue</a>" ordata-footerhtml="#some_other_id".

<button type="button" 
  data-toggle="modal" 
  data-title="This is the title of the modal"
  data-titledescription="description of title"
  data-bodyhtml="#some_id"
  data-footerhtml="#some_other_id"
>Launch modal</button>

Via JavaScript

You can call a modal with id myModal within a JavaScript function:

Lets say you have:
<a href="#" class="btn btn-secondary" id="launchMyModal">Launch my modal</a>

A javascript function would be used to handle the click event:

$("#launchMyModal").click(function(event){
  event.preventDefault();
  $('#myModal').modal('show');
})

Explanation:
$("#launchMyModal") refers to the element that is the trigger to launching the modal.
event.preventDefault(); prevents the browser from actually attempting to go to #myModal as it would normaly would.
$('#myModal').modal('show'); is what actually activates the modal.
('show') is one of a number of methods detailed below.


Tip: 
<div id="myModal" ... should have a class="hide" so that it remains hidden until it is activated.

Generic modal via JavaScript

You can call the generic modal with within a JavaScript function:

Lets say you have:
<a href="#" class="btn btn-secondary" id="launchMyModal">Launch my modal</a>

A javascript function would be used to handle the click event:

$("#launchMyModal").click(function(event){
  event.preventDefault();
  Sherpa.publish("modal", {
    title:'Title of Modal (optional)',
    title_description:'description that goes under title (optional)',
    body_html: '#some_id',
    footer_html: '#some_other_id',
  })

})

Explanation:
$("#launchMyModal") refers to the element that is the trigger to launching the modal.
event.preventDefault(); prevents the browser from actually attempting to go to #myModal as it would normaly would.
Sherpa.publish("modal" activates the generic modal and lets you pass configuration attributes which match the same as the html attributes described above.


Tip: 
<div id="some_id" and <div id="some_other_id" used for the body and footer should have a class="hide" so that they remain hidden.

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-backdrop="".

Name type default description
backdrop boolean true Includes a modal-backdrop element. Alternatively, specify static for a backdrop which doesn't close the modal on click.
keyboard boolean true Closes the modal when escape key is pressed
show boolean true Shows the modal when initialized.
remote path false

If a remote url is provided, content will be loaded via jQuery's load method and injected into the .modal-body. If you're using the data api, you may alternatively use the href tag to specify the remote source. An example of this is shown below:

  1. <a data-toggle="modal" href="remote.html" data-target="#modal">click me</a>

Methods

.modal(options)

Activates your content as a modal. Accepts an optional options object.

  1. $('#myModal').modal({
  2.   keyboard: false
  3. })

.modal('toggle')

Manually toggles a modal.

  1. $('#myModal').modal('toggle')

.modal('show')

Manually opens a modal.

  1. $('#myModal').modal('show')

.modal('hide')

Manually hides a modal.

  1. $('#myModal').modal('hide')

Events

Bootstrap's modal class exposes a few events for hooking into modal functionality.

Event Description
show This event fires immediately when the show instance method is called.
shown This event is fired when the modal has been made visible to the user (will wait for css transitions to complete).
hide This event is fired immediately when the hide instance method has been called.
hidden This event is fired when the modal has finished being hidden from the user (will wait for css transitions to complete). 
  1. $('#myModal').on('hidden', function () {
  2.   // do something…
  3. })
