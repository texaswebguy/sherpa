File structure

Within the download you'll find the following file structure and contents, logically grouping common assets.

Once downloaded, unzip the compressed folder to see the structure of (the compiled) Sherpa. You'll see something like this:

  sherpa/
  ├── .sherpa/
  ├── assets/
  │   ├── config/
  │   ├── content/
  │   ├── css/
  │   │    ├──doc.css
  │   ├── data/
  │   ├── img/
  │   ├── js/
  │   │    ├──app.js
  │   ├── xref/
  ├── includes/
  │   ├── sample/
  ├── pages/
  │   ├── templates/
  ├── index.html
  ├── launch_prototype.bat
  ├── project_index.html
  ├── site.html

Please review the explanation of files and folders below:

  • .sherpa: Here all framework support files are stored.

  • assets: Here all prototype support files are stored.

  • assets/config: Config is a folder where configuration files for each prototype application is stored in for of JavaScript (.js) files.

  • assets/content: Contains messages files that are in JSON format. All files follow the same naming convention which is messages_locale.json. locale can be a language or local code such as""en" or "en_US" respectively.

  • assets/css: These are the project specific css files. You will see sherpa_overrides and bootstrap files. Please ignore these. They are here because of legacy purposes but will be removed eventually. The key document here is doc.css which by default it automatically gets loaded with every prototype page.

  • assets/data: A place where sample data can be stored in any format such as json, csv, etc.

  • assets/img: General location for all project specific images. The images in this directory are related to the Sherpa website. They will eventually be removed so that the folder is completely empty and only have project specific images.

  • assets/js: General location for all project JavaScript files. You should notice a sherpa-site.js file that is related to the Sherpa website. This file will eventually be removed so that the folder is completely empty and only have project specific JavaScript files. app.js automatically gets loaded with every prototype page. app.js contains sample data that will be eventually deleted.

  • assets/xref: Contains cross reference files that are in JSON format. All files follow the same naming convention as messages like resources_locale.json. locale can be a language or local code such as""en" or "en_US" respectively. These files are used to provide link and image references.

  • assets/sherpa: This is the core files for the Sherpa prototyping framework. When updating the core to a new release, this is the folder that would bereplaced.

  • components: All established components will be in this forlder. No project work should be done in this folder unless the work is to improve or create established components.

  • includes: Very similar to components but project specific. It is possible that an include in a particular project be made into a component.

  • pages: These are virtual pages that can be organized in any number of folders. Notice the templates folders which has page definitions that help you achieve certain layouts. These can be copied into other folders and be used.

  • index.html: index.html and site.html are exactly the same. site.html is used for when index.html is wre writen from project_index.html

  • launch_prototype.bat: Batch file to initiate the web server and launch the prototype index page.

  • project_index.html: project_index.html can be renamed as index.html
