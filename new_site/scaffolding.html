Global settings

Requires HTML5 doctype

Sherpa makes use of certain HTML elements and CSS properties that require the use of the HTML5 doctype. Include it at the beginning of all your projects.

<!DOCTYPE html>
  ...
</html>

Typography and links

Sherpa sets basic global display, typography, and link styles. Specifically, we:

  • Remove margin on the body
  • Set background-color: white; on the body
  • Use the body attribute to set the typographic base
  • Use the h1 - h5 attributes as our header typographic base

These styles can be found within sherpa_override.css.

Reset via Normalize

Sherpa uses Normalize.css, a project by Nicolas Gallagher that also powers the HTML5 Boilerplate. Normalize.css makes browsers render all elements more consistently and in line with modern standards. It precisely targets only the styles that need normalizing.

Responsive Classes

Sherpa uses head.js to manage the loading of all javascript and css. Using head.js Sherpa adds responsive css classes to the html element which can help

The following classes are automatically assigned to the html element when the page loads and resizes:

  • desktop
  • no-desktop
  • tablet
  • no-tablet
  • mobile
  • no-mobile

Device Specific Classes

The following classes are automatically assigned to the html element when the page loads:

  • touch
  • no-touch
  • retina
  • no-retina

Device Orientation Specific Classes

The following classes are automatically assigned to the html element when the page orientation changes:

  • landscape
  • no-lansdscape
  • portrait
  • no-portrait

Browser Specific Classes

The following classes are automatically assigned to the html element when the page loads:

  • borderimage: define an image to be used instead of the normal border of an element
  • borderradius: support for rounded borders
  • boxshadow: a simple shadow for a box
  • cssreflections: a reflection effect for an object
  • csstransforms: ability to scale
  • csstransitions: ability to animate changes to CSS properties
  • fontface: ability to use custom fonts
  • multiplebgs: ability to define multiple background properties for a box
  • opacity: transparency support
  • rgba: allows to specify an opacity value for a color
  • textshadow; a simple shadow for a text

a "no-" version shows when above items are not supported

Localization attribute

Based on the language/locale which the prototype is set for the "lang" attribute is automatically added to the html element.

Pseudo Authentication

Sherpa has a pseudo authentication method that allows you to easily mock a logged-in scenario using two css classes.

  • authenticated-only: Only displays when the prototype is :"authenticated"
  • not-authenticated-only: Only displays when the prototype is NOT :"authenticated"

See functions to learn ways to trigger the pseudo authentication.

Grid overview

Create multi-device layouts quickly and easily with the nestable grid. We have (6) different grids within the set that apply to a mobile (2), tablet (2), desktop (2) site layout.

Live grid example

This example shows different combinations of layouts and how they change based on screen size.

Desktop grid

The desktop grid utilizes 12 columns and 15 columns with a max-width of 1140px and a min-width of 980px.

Desktop (B) - 15 columns

The 15 column grid uses a class of .db for columns.

Tablet (A) - 9 columns

The 12 column grid uses a class of .ta for columns.

Tablet (B) - 8 columns

The 12 column grid uses a class of .tb for columns.

The mobile grid utilizes 3 columns and 4 columns with a max-width of 767px.

Mobile (A) - 3 columns

The 3 column grid uses .ma for columns.

Mobile (B) - 4 columns

The 12 column grid uses .mb for columns.

Basic grid HTML

For a simple two column layout, create a .wrapper and add the appropriate number of columns. Remember that you have to add the grid classes for desktop .da or .db, tablet .ta or .tb and mobile .ma or .mb.

Your HTML should look something like this:

<div class="wrapper">
  <div class="da1-da3 ta1-ta3 ma-all">...</div>
  <div class="da4-da12 ta4-ta9 ma-all">...</div>
</div>

For desktop we have .da1-da3 and .da4-da12, making for 12 total columns and a complete row. For tablet we have .ta1-ta3 and .ta4-ta9, making for 9 total columns and a complete row. For mobile we have .ma-all for both, making for 3 total columns and two complete row.

Using the grid

The grid system will generate all the classes you need to layout your page. For each grid, use the prefix plus the columns you want to span to define the width and margins of an element. So an element that needs to span columns 3 to 5 on your desktop grid would get a class of da3-da5 (where da is your defined prefix for the desktop grid). An element that only spans the second column would be da2.

If that same element needs to span columns 1 to 2 on your tablet grid, just add a class like so: ta1-ta2 to your da3-da5.

Example code

<section>
 <div class="wrapper">
  <article>
   <div class="da1-da3 ta1-ta3 ma1">I will span the first to third columns of the desktop (a) grid, the 
   first to third columns for the tablet (a) grid, and the first column for the mobile (a) grid.</div>
  </article>
 </div>
</section>

Offsets

The grid CSS calculates all offsets for you. There is no need to worry about how many columns should be spaced between elements: this is all automatically applied by the CSS. You only need to add the right class to the right element.

Predefined Classes

For each grid, you also get some predefined classes that will help you with tricky layout situations. In these examples, * is the prefix defined for your grid:

  • *-all

    Span the full width of the grid. Helps with clearing, too. Equivalent of adding width:100%.

  • *-hide

    Hide the element for this grid.

  • *-clear

    Clear left and offset by the distance from column one to the first column the element spans.

  • *-overlap

    Adds clear:none to an element for a grid. Use this class to place elements without being limited by source order.

  • *-floatright, *-floatleft

    Floats the element in a direction, adding the proper gutter margin to the opposite side. Allows text to wrap element.

  • *-clear

    Clear left and offset by the distance from column one to the first column the element spans.

  • *-pad

    Add left and right padding of half the gutter width on the outside of the element.

  • *-padfull

    Add left and right padding of the full gutter width on the outside of the element.

  • *-padin

    Add left and right padding of half the gutter width on the inside of the element.

  • *-padinfull

    Add left and right padding of the full gutter width on the inside of the element.
